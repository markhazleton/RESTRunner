## Core Principles

### I. Layered Domain-First Architecture (MANDATORY)

The codebase is organized around a stable domain layer, explicit service interfaces, and separate execution and UI hosts. New work must preserve that separation unless there is a documented exception.

- Domain models and contracts MUST remain separate from transport, storage, and UI concerns.
- Execution logic MUST live behind service interfaces and dependency injection boundaries.
- Console and web entry points MUST compose services rather than absorb business logic.

Evidence:
- RESTRunner.Domain contains shared models and interfaces used by the other projects.
- RESTRunner.Services.HttpClient/ExecuteRunnerService.cs implements IExecuteRunner for the execution engine.
- RESTRunner.Web/Program.cs wires interfaces to file-backed and execution services.

- Projects MUST target the current solution framework version consistently.

### III. MSTest Is The Standard Test Framework (MANDATORY)

Automated tests currently use MSTest conventions in the dedicated test project, and that should become the explicit standard.

- Automated tests MUST use MSTest unless a deliberate replacement is approved.
- Test files SHOULD live in dedicated test projects and use the existing *Tests.cs naming pattern.

- Test classes and methods consistently use TestClass and TestMethod attributes in RESTRunner.Domain.Tests.

The repository already distinguishes between application logging and console reporting. That distinction should be kept explicit rather than mixed arbitrarily.
